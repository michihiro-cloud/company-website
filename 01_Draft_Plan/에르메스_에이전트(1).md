## 📋 1. 프로젝트 개요  
**목표** –  
- **회사 홈페이지**(제품·서비스 소개, 포트폴리오, 연락처 등) 를 **VS Code** + **Claude Code** 로 코딩하고, **Obsidian Vault** 로 기획·콘텐츠·RAW 데이터를 관리한다.  
- 모든 파일을 **GitHub**에 올려서 CI/CD(자동 배포) 를 구현한다.  
- **초보자**도 따라 할 수 있도록 단계별 가이드와 샘플 코드를 제공한다.  
  
> **핵심 흐름**  
1. **Obsidian**에 기획·콘텐츠·이미지·데이터(마크다운) → 로컬 폴더에 저장  
2. 동일 폴더를 **VS Code** 로 열어 **Claude Code**에게 “이 마크다운을 HTML/React 로 구현해줘” 라는 프롬프트 전송 → 코드 자동 생성  
3. 생성된 코드를 **Git** 으로 커밋 → **GitHub**에 푸시 → **Vercel/Netlify** 로 자동 배포

## 📁 2. 폴더·파일 구조 (Obsidian Vault ↔ VS Code)  
  
```  
my-company-website/ # ★ VS Code + Git 루트 / Obsidian Vault 루트  
│  
├─ .github/ # GitHub Actions (CI/CD) 설정  
│ └─ workflows/  
│ └─ deploy.yml  
│  
├─ docs/ # Obsidian에 쓰는 원본 마크다운 (기획·콘텐츠)  
│ ├─ 00_개요.md  
│ ├─ 01_회사소개.md  
│ ├─ 02_서비스_스테레오3D.md  
│ ├─ 03_기술스택.md  
│ ├─ 04_포트폴리오.md  
│ └─ 05_연락처.md  
│  
├─ public/ # 정적 파일 (이미지·favicon·favicon.ico 등)  
│ └─ images/  
│ ├─ logo.png  
│ └─ hero.jpg  
│  
├─ src/ # Claude Code가 생성·수정할 코드  
│ ├─ components/  
│ │ ├─ Header.jsx  
│ │ ├─ Footer.jsx  
│ │ └─ ServiceCard.jsx  
│ ├─ pages/  
│ │ └─ index.jsx  
│ └─ App.jsx  
│  
├─ .gitignore  
├─ README.md  
├─ package.json  
├─ vite.config.js # Vite(또는 Astro, Next.js 등) 설정  
└─ obsidian-sync.sh # Obsidian ↔ VS Code 동기화 스크립트 (선택)  
```  
  
- **Obsidian** → `docs/` 폴더에 마크다운을 그대로 저장합니다.  
- **VS Code**는 같은 루트를 열어(`my-company-website`) 작업하므로 두 툴이 **실시간으로 같은 파일을 공유**하게 됩니다.  
  
> **Tip**: Obsidian 플러그인 *Obsidian Git* 을 켜두면 `docs/` 폴더를 자동 커밋/푸시 할 수 있어 언제든 최신 기획을 GitHub에 반영할 수 있습니다.

## 📦 4. 프로젝트 초기화 (VS Code ↔ Obsidian)  
  
```bash  
# 1️⃣ 프로젝트 루트 생성  
mkdir my-company-website && cd my-company-website  
  
# 2️⃣ npm 초기화 + Vite‑React 템플릿 (초보자용)  
npm create vite@latest . -- --template react  
  
# 3️⃣ 필요한 패키지 설치  
npm install  
  
# 4️⃣ Git 초기화 & GitHub 원격 저장소 연결  
git init  
gh repo create stereo3d‑company‑site --public --source=. --remote=origin  
git add .  
git commit -m "Initial commit: Vite‑React skeleton"  
git push -u origin main  
```  
  
### Obsidian에 같은 폴더 연결  
  
1. **Obsidian → Settings → Vault → Open folder as vault** 선택  
2. 방금 만든 `my-company-website` 폴더를 선택 → **Vault 이름**을 `Stereo3D‑Site` 로 지정  
3. 이제 **Obsidian → Docs** 폴더 안에서 마크다운을 작성하면 VS Code에서도 바로 보입니다.

## 🤖 5. Claude Code와 VS Code 연동  
  
### 5‑1. Claude Extension 설정 (VS Code)  
  
1. **VS Code → Extensions** → “Claude” 검색 → 설치  
2. `Ctrl+Shift+P` → “Claude: Sign In” → API 키 입력 (위에서 export 한 키가 자동 인식)  
  
### 5‑2. Claude에게 “마크다운 → 코드” 요청 예시  
  
> **예시 프롬프트** (Obsidian에 `01_회사소개.md` 가 있다고 가정)  
  
```markdown  
# 01_회사소개.md (Obsidian)  
  
## 우리 회사  
Stereo 3D는 AI‑기반 S3D(스테레오 3D) 컨버팅 서비스를 제공하는 기업입니다.  
...  
```  
  
**Claude Code**에 전달할 프롬프트:  
  
```  
/** Claude 명령 */  
convert docs/01_회사소개.md to a React component.  
- Create a file src/pages/About.jsx  
- Use Tailwind CSS for styling  
- Insert the markdown content as JSX (preserve headings, paragraphs, lists)  
- Export default About  
```  
  
VS Code **Command Palette**(`Ctrl+Shift+P`) → **Claude: Send Prompt** 입력 → 위 텍스트 붙여넣기 → 실행  
  
Claude가 `src/pages/About.jsx` 파일을 자동으로 생성·삽입합니다.  
  
> **팁**: `// @claude` 같은 주석을 파일에 넣어 두면 “다음에 이 파일을 수정해줘” 라는 지시를 간단히 전달할 수 있습니다.
