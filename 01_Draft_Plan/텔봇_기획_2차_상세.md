**2️⃣ Obsidian Vault 공유 활성화**  
  
Obsidian에서 Vault를 다른 앱에서 접근 가능하게 하려면:  
  
**방법 A: 로컬 폴더 직접 접근 (가장 간단)**  
- Obsidian Vault는 기본적으로 로컬 폴더에 저장됨  
- VS Code에서 해당 폴더를 열기만 하면 Claude Code가 접근 가능  
  
**방법 B: Obsidian Sync / 공유 활성화**  
- Obsidian 설정 → **파일 및 링크** → **Vault 경로 확인**  
- 기본 경로 예시:  
- Windows: `C:\Users\<사용자>\Documents\MyVault`  
- macOS: `~/Documents/MyVault`  
- Linux: `~/ObsidianVault`  
  
---  
  
**3️⃣ VS Code에서 Vault 폴더 열기**  
  

VS Code → File → Open Folder → Obsidian Vault 폴더 선택

  
  
이렇게 하면:  
- `.md` 파일들이 VS Code에서 바로 보임  
- Claude Code가 Vault 내 모든 노트를 읽고 쓸 수 있음  
  
---  
  
**4️⃣ Claude Code로 Obsidian 노트 활용**  
  
VS Code에서 Claude Code를 열고 다음과 같이 활용:  
  
**📖 노트 검색 & 요약**  

"Vault에서 '프로젝트 기획' 관련 노트 찾아줘"
"~/ObsidianVault 폴더의 모든 노트 요약해줘"

  
  
**✍️ 노트 생성 & 편집**  

"프로젝트 회의록을 마크다운으로 작성해서 
 ~/ObsidianVault/Meetings/2026-06-19.md로 저장해줘"

  
  
**🔗 백링크 & 그래프 활용**  

"이 노트와 연결된 다른 노트들을 찾아줘"
"[[프로젝트A]] 태그가 있는 모든 파일 목록 보여줘"

  
  
**💡 코딩에 활용**  

"Vault의 API 문서 노트를 참고해서 
 Python 클라이언트 코드 작성해줘"
"~/ObsidianVault/Docs/database-schema.md 내용을 
 바탕으로 SQL 마이그레이션 스크립트 만들어줘"

**실전 활용 시나리오:**  
  
1. **문서 기반 코딩**: Vault의 기술 문서를 컨텍스트로 제공 → Claude Code가 문서 기반으로 코드 생성  
2. **회의록 → 코드**: 회의록 노트를 읽고 → 액션 아이템을 코드 태스크로 변환  
3. **지식 베이스 활용**: 이전 프로젝트 노트를 참고 → 유사 문제 해결  
4. **코드 → 문서화**: 작성한 코드를 자동으로 Obsidian 노트로 문서화

****Claude Code에 경로 알려주기****  
• 설명: `~/MyVault` 형태로 정확한 경로 지정  
  
****Obsidian Live Preview****  
• 설명: VS Code에서도 Markdown 미리보기 가능 (`Ctrl+Shift+V`)

**8단계: 인터넷에 배포하기**  
  
**무엇을 하는가?**  

> 완성된 홈페이지를 인터넷에 올려서 누구나 볼 수 있게 합니다.  

  
**방법: GitHub Pages (무료, 가장 쉬움)**  
  
**① GitHub 가입**  
1. https://github.com 접속  
2. 회원가입 (무료)  
  
**② 저장소(Repository) 만들기**  
1. GitHub에서 **"New repository"** 클릭  
2. 이름: `company-website`  
3. **Public** 선택  
4. **Create repository** 클릭  
  
**③ 코드 업로드**  
VS Code 터미널에서:  
  

cd ~/CompanyWebsite/src
git init
git add .
git commit -m "첫 번째 홈페이지"
git branch -M main
git remote add origin https://github.com/내아이디/company-website.git
git push -u origin main

  
  

> **용어 설명:**  
> - git = 코드 버전 관리 도구  
> - commit = 변경사항 저장  
> - push = 온라인에 업로드  

  
**④ GitHub Pages 활성화**  
1. GitHub 저장소 페이지에서 **Settings** 클릭  
2. 왼쪽 메뉴에서 **Pages** 클릭  
3. **Source**에서 **main** 브랜치 선택  
4. **Save** 클릭  
5. 몇 분 후 `https://내아이디.github.io/company-website` 에서 확인
